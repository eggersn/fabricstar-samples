4. First-Network Overview

4.1 Vorgenommene Aenderungen:
Das Netzwerk besteht aus:
- 3x Zookeeper (docker-compose-kafka.yaml)
- 4x Kafka broker (docker-compose-kafka.yaml)
- 1x Orderer (docker-compose-cli.yaml)
- 2x2 Peer Nodes (docker-compose-cli.yaml)
	i.e. 2 Peers pro Organisation
- 1x Client

configtx.yaml:
Hier wird eingestellt welche Art von Orderer genutzt wird (in unserem Fall Kafka)

Anmerkungen: Für die Kafka broker wird das Image fabric_star/kafka lokal benötigt. D.h. zunächst muss das Kafka Projekt von Marvin compiled werden und anschließend das build_docker.sh Skript ausgeführt werden.

5. Ausführung:
".\byfn.sh up -o kafka" erstellt alle genannten Docker Container und führt ein paar Queries und Invokes durch.
Mit "docker exec -it cli bash" wird eine shell in dem Client Container gestartet.
Damit können dann insbesondere die auf
https://hyperledger-fabric.readthedocs.io/en/release-1.4/build_network.html
beschriebenen Commands individuell getestet werden.

Um die Logs für Debugging Möglichkeiten einzusehen ist der Command
"docker logs -f <container>" hilfreich. Insbesondere kann mit
"docker logs -f kafkaX.example.com" für X = 0,1,2,3 geprüft werden,
ob der Orderer sich mit einem Kafka broker verbindet. Beim Ausführen sollte
für mindestens einen der broker die von Marvin eingefügten Meldungen ähnlich zu
"INFO [KafkaApi-3] !!!!!!!!!!!! PartitonData: (offset=2, logStartOffset=-1, maxBytes=1048576) (kafka.server.KafkaApis)"
erscheinen.
